# Launch

## Background

Launch is moving from closed testing to production on Google Play. It's not the finish line — it's where the real work starts.

---

## Do this

### Before launch

- [ ] 100+ testers in closed testing
- [ ] Major bugs fixed
- [ ] Store listing ready
- [ ] Screenshots polished
- [ ] Privacy policy URL live

### Store listing

**App name:** Short, includes keyword, no jargon.

**Description:**
```
Line 1: What it does (hook)
Line 2-3: Key features
Line 4: CTA
```

**Screenshots:** Show the app in use. Add text overlays. First screenshot = most important.

### ASO (App Store Optimization)

| Element | Impact |
|---------|--------|
| App name | High |
| Screenshots | High |
| Description | Medium |
| Reviews | High |

### Launch day

- [ ] Submit for review
- [ ] Prepare social posts
- [ ] Tell your testers
- [ ] Post in niche communities
- [ ] Monitor crash reports

### After launch

- [ ] Respond to reviews
- [ ] Fix critical bugs fast
- [ ] Plan next update
- [ ] Track retention

---

## Evidence

We shipped multiple apps from closed testing to production with 100-140 testers each. All approved without issues. The key was having enough testers and fixing the obvious bugs before submitting.

---

*Launch is the beginning, not the end.*
